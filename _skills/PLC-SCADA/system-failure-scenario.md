# System Failure Scenario & Simulation

## Copy/Paste This Prompt

---

I need to analyze a system failure scenario or simulate what would happen:

**SYSTEM:** [What system? What does it do?]

**SYSTEM COMPONENTS:**
- [Component 1: function/specs]
- [Component 2: function/specs]
- [Component 3: function/specs]
- [PLC/SCADA logic: describe briefly]
- [Safety systems: describe]

**SCENARIO:** [What fails or what condition occurs?]

**FAILURE/CONDITION DETAILS:**
- What fails: [Specific component/function]
- When: [Immediate/gradual/under load/etc.]
- Cause: [Known or to be determined]
- Current state: [What's happening now]

**QUESTIONS TO ANSWER:**
1. What happens to the rest of the system?
2. Will it shut down safely or continue operating dangerously?
3. What's the impact on operations?
4. What's the safety impact?
5. How long can it continue in this state?
6. What should happen instead?
7. What's the best response?

**CONSTRAINTS/ASSUMPTIONS:**
- [Constraint 1: e.g., "Operator is aware of the failure"]
- [Constraint 2: e.g., "System must continue running"]
- [Constraint 3: e.g., "No external support available"]

**APPLICABLE STANDARDS:**
- [IEC standard / Safety standard / Other]

Please simulate:
- Immediate effects of the failure
- Cascading failures or secondary impacts
- System state progression
- Safety implications
- Operational consequences
- Failure detection and response
- What should happen vs. what will happen
- Time-critical issues
- Recovery options

Provide:
- Scenario progression timeline
- Step-by-step system behavior
- Safety assessment
- Operational impact assessment
- Detection of failure (manual/automatic)
- Emergency response adequacy
- Recommendations to prevent or mitigate
- Suggested system improvements

## Notes

- Describe the scenario clearly and specifically
- Include all relevant system details
- Note any known limitations or weak points
- Include what you think might happen
- Mention any previous similar incidents
